# Patient API

## Описание проекта
Этот проект представляет собой API для управления данными пациентов. Он позволяет создавать, обновлять, удалять и получать информацию о пациентах.

## Тестирование
Вы можете тестировать API с помощью Docker и Swagger. 

### Запуск через Docker
При запуске приложения через Docker, API будет доступно по следующему URL:
- **URL**: `http://localhost:8080/index.html`

### Порты
- **HTTP порт**: 8080

## Документация
В папке `Docs` находится коллекция для Postman, которая содержит примеры запросов для работы с API. Вы можете импортировать эту коллекцию в Postman для удобного тестирования.

## Консольное приложение
В проекте также имеется консольное приложение, которое используется для добавления 100 пациентов в базу данных. Это приложение упрощает процесс тестирования и демонстрации функциональности API.

## Управление пациентами
- **Создание пациента**: При создании нового пациента его `Id` генерируется автоматически. Однако, этот `Id` не возвращается в ответе, поэтому для получения информации о пациенте по `Id` необходимо использовать методы поиска по дате рождения.
  
- **Удаление пациента**: При удалении пациента он не удаляется из базы данных. Вместо этого, поле `Active` устанавливается в `false`, что позволяет сохранить информацию о пациенте, но исключить его из активных записей.

## Поиск пациентов
Для поиска пациентов по дате рождения реализованы различные методы. Вы можете использовать следующие варианты:

1. **Поиск по дате рождения (больше чем)**: Позволяет найти пациентов, родившихся после указанной даты.
2. **Поиск по дате рождения (меньше чем)**: Позволяет найти пациентов, родившихся до указанной даты.
3. **Поиск по дате рождения (больше или равно)**: Позволяет найти пациентов, родившихся в указанную дату или позже.
4. **Поиск по дате рождения (меньше или равно)**: Позволяет найти пациентов, родившихся в указанную дату или раньше.

Для получения списка пациентов по дате рождения используйте соответствующие методы API.
